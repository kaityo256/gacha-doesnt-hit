# ガチャの沼へようこそ

「ガチャ」という言葉を聞いたことがあるだろう。もともとは「カプセルトイ」と呼ばれる小型の自動販売機に由来する。カプセルトイは硬貨を入れてレバーを回すと、カプセルに入ったおもちゃなどが出てくるものだ。レバーを回す際にガチャガチャ音がなるので、ガチャガチャなどと呼ばれるようになった。バンダイは「ガシャポン」や「ガチャガチャ」という名前で商標登録をしている。お金を入れてカプセルが出てくるまで何が出てくるかわからないという楽しみがあり、たまに「シークレット」と呼ばれる、表に表示されていない種類が入っていることがあり、何度もお金を入れたくなる装置である。

そのうち、特にソーシャルゲーム(ソシャゲ)などで、購入した時にもらえるアイテムがランダムで決まるようなものも「ガチャ」と呼ばれるようになった。アイテムには「ノーマル」「レア」「スーパーレア」「ウルトラレア」などと「レア度」が決まっており、レアなアイテムを求めて何度も課金してしまう、いわゆる「廃課金」が問題となった。以下では、そのようなガチャについてのシミュレーションしてみよう。

## アイドルのイベント

あるアイドルが新曲のリリースを記念してイベントを行った。そのシングルCDの購入者に対してポスターをプレゼントする。ポスターは全部で44種類あり、もらえるポスターはランダムである。このポスターの44種類すべてコンプリートしたら、特別なイベントに招待します、というものだ。最初にCDを購入した時、ポスターが1枚もらえる。2枚目を購入した時に、同じポスターを当ててしまう確率は44分の1だ。手元のポスターの書類が増えるほど、次にもらえるポスターが手持ちと「かぶる」可能性が高くなっていく。例えば、いま手元に22種類のポスターを持っているなら、次のCDがかぶる確率は50%だ。さて、ポスターをすべてコンプリートするには、平均で何枚のCDを購入すればよいだろうか？答えを読む前に、まず予想してみてほしい。予想できただろうか？では次に進もう。

この問題は厳密解が存在するが、まずはシミュレーションしてみよう。まずは1から44の数字を紙に書いておく。44つの目があるサイコロを用意して、サイコロを振っては出た目の数字を消す。サイコロを振った数と、消した数字の数の関係を調べ、最後の数字を消した時の「サイコロを投げた回数」を記録すればよい。コードも非常に短いので、以下に書いてみる。

```py
import random

total = 0
for i in range(10000):
  cd = []
  n = 0
  while len(set(cd)) < 44:
    i = random.randint(1,44)
    cd.append(i)
    n += 1
  total += n

print(total/10000)
```

実行してみると、毎回結果が変わるが、およそ190～193という値が出るであろう。
