# リボ払いの恐怖

## 数値シミュレーションとは

本書のテーマは数値シミュレーションである。数値シミュレーションはシミュレーションの一種だ。シミュレーションとは「予め定めたルールに従う世界の振る舞いを再現、予想すること」である。例えば都市や国を運営するゲームはシミュレーションゲームと呼ばれるジャンルである。飛行機のパイロットは、フライトシミュレーターで訓練を行う。フライトシミュレーターは実機と同じ計器や操縦桿が用意され、同じ感覚で操縦できる。例えば悪天候での飛行や、エンジントラブルなど危険な状況を再現し、操縦やトラブル対応の訓練ができる。避難訓練などもシミュレーションの一種だ。

数値シミュレーションとは、シミュレーションを数値的に行うことだ。

## リボ払いのシミュレーション

数値シミュレーションの例として、リボ払いについて考えてみよう。リボ払いとは、毎月の支払額を一定の金額に固定して借金を返済する支払い方法だ。支払い方式は主に「元利定額方式」と「残高スライド方式」があるが、以下では「元利定額方式」について考えてみよう。なお、これから説明する内容はかなり簡略化されており、実際の返済方式とは異なる場合があることに注意。

元利定額方式とは、返済手数料を含めた毎月の支払金額が一定となる方式だ。借金残高とは無関係に毎月の支払額が固定されるため、計画が立てやすい。月々の支払い金額から借金残高に対応する金利手数料を除いた額だけ返済に充てられ、借金が減っていくことになる。

例えば、金利が年率15%であるとしよう。毎月の支払額を1万円に固定して、10万円の借金をしたとする。年率15%であるから、10万円を1年間借りた時の利息は15000円だ。毎月の利息は日割り計算とすることが多いが、簡単のために年率を12等分したものを月の利息としよう。すると、1250円が初月の金利手数料となる。毎月の支払額1万円のうち、1250円が金利手数料、残りの8750円が残高返済に充てられる。次の月は、借金残高が91250円となっているため、金利手数料が1140円、残高返済が8860円となる。このように、毎月の返済金額は同じ1万円だが、借金残高が減るにつれて残高返済に充てられる金額が増えていき、借金が減る速さが早くなっていく。最終的に、11回の支払い、総額10万7497円で借金の返済が完了する。この結果を見て「あれ？思ったより安いな？」と思わなかっただろうか？


これは「ルールは事前に完全にわかっているが、結果は非自明」である例になっている。
