# ドアの向こうは運か戦略か？

確率の問題は、しばしば直感に反する結果をもたらす。本稿では、そのような問題の例をいくつか取り上げるとともに、乱数を用いたシミュレーションについても見ていこう。

## モンティ・ホール問題

こんなゲームを考えよう。ゲームの挑戦者の前には三つの箱があり、一つがアタリ、残りがハズレだ。アタリの箱を選べば景品がもらえる。司会者は挑戦者にどれか一つを選ぶように言う。挑戦者が一つ箱を選ぶと、司会者は挑戦者に見えないように残りの二つの箱の中身を確認し、ハズレの箱を開ける。ここで、司会者は挑戦者に「最初に選んだ箱のままでも良いし、残っている開けられていない箱を選んでも良い」と言う。挑戦者が「選択を変える」か「変えない」かを選んだあと、箱を開けて挑戦者が最終的に選択した箱がアタリかハズレかを確認する。さて、あなたは最初に選んだ箱を変更するべきであろうか？それとも変更する、しないに関わらず景品が当たる確率は変わらないだろうか？

![モンティ・ホール問題](fig/montyhall.png)
モンティ・ホール問題。三つの箱のうち、当たりの箱は一つだけ。(1) 挑戦者は最初に一つ箱を選ぶ。(2) 司会者は選ばれなかった二つの箱のうち、ハズレの箱を開ける。(3) 挑戦者は、残ったもう一つの箱を選び直すチャンスが与えられる。

これはモンティ・ホール問題と呼ばれる。もともとアメリカのテレビ番組のコーナーの一つで、その後雑誌のコラムにこの問題が取り上げられ、多くの有識者が答えを間違ったことで話題となった。有名な問題なので答えを知っている人もいるであろうが、なぜそれが正しいかを説明できるだろうか？

さて、この問題は確率の問題である。数学のテストで答を記入した後、それが正しいか不安な時に「見直し」をすると思うが、確率の問題は「見直し」が難しいという性質がある。例えば方程式の問題なら、得られた答えを最初の式に代入することで正しいかどうかが判定できる。漸化式の一般項を求める問題も、最初の数項を調べて見れば、少なくとも間違っている時にはすぐに分かるし、最初の三項の値が正しければ、まず正解であろう。

一方、確率の問題はどうだろうか。答えが出ても何かに代入して確認したりできないし、間違えて解答を読んでその時には納得した気がしても、なんか騙された気がしてモヤモヤする、そんな気持ちになったことがあるだろう。そんな時にシミュレーションをすることで簡単に正しい答えを推定することができる。数学的に考えてみる前に、まずはシミュレーションをしてみよう。

シミュレーションのアルゴリズムを考える。箱を3つ用意して、それぞれに1,2,3と番号をつけよう。ゲーム開始時に正解の箱をランダムに選んでおく。さて、挑戦者も最初に箱をランダムに選ぶ。この後、司会者が残りの箱からハズレの箱を開けた時に、挑戦者は「最初の選択を変えない」か「残りの箱に変える」か、二つの選択をすることができる。ここで「常に選択を変えない挑戦者」をキープ派、「常に選択を変える挑戦者」をチェンジ派と呼ぼう。キープ派とチェンジ派、繰り返しゲームをさせることで、どちらが景品を当てる確率が高いかを求めることができる。

さて、キープ派のプログラムは簡単だ。選択を変えないのだから、最初に選んだ箱がアタリならアタリ、そうでなければハズレである。

```py
from random import choice
games = 10000
wins = 0
for _ in range(games):
    boxes = [1,2,3] # 三つの箱を用意する
    answer = choice(boxes) # 正解の箱を決める
    first_choice = choice(boxes) # 最初の箱を選ぶ
    # 正解と一致していたらアタリ
    if first_choice == answer:
        wins = wins + 1
print(wins/games)
```

図1：キープ派の景品獲得確率計算コード

このコードを実行すると、結果は毎回変わるが、例えば0.3286や、0.3277といった数値が表示される。これを見ると、「およそ1/3なんだろうな」と予想がつく。考えてみれば、最初に選んだ箱がアタリならアタリ、そうでなければハズレなのだから、確率は1/3になるはずだ。


一方、チェンジ派の場合はすこしややこしい。司会者は、挑戦者が箱を選んだ後、残りの二つを挑戦者に見えないように確認し、ハズレの箱を開けるのであった。挑戦者が最初に正解の箱を選んでいたら、司会者は残りの二つからランダムに開ける箱を決める。一方、挑戦者が最初に不正解の箱を選んでいたら、残りの二つの箱のどちらかは正解であるから、正解でない方を開ける必要がある。そして、チェンジ派は「司会者が開けなかった方の箱」を選ぶ。以上をそのままコードにすると図2のようになる。

```py
from random import choice
games = 10000
wins = 0
for _ in range(games):
    boxes = [1,2,3] # 三つの箱を用意する
    answer = choice(boxes) # 正解の箱を決める
    first_choice = choice(boxes) # 最初の箱を選ぶ
    # 最初に選んだ箱がアタリかどうか
    if first_choice == answer: 
        # アタリなら、残りの二つからランダムに選ぶ
        boxes.remove(answer)
        second_choice = choice(boxes)
    else:
        # ハズレなら、残りは必ず正解
        second_choice = answer
    if second_choice == answer:
        wins = wins + 1
print(wins/games)
```

図2: チェンジ派の景品獲得確率計算コード

これを実行すると、0.6694や0.6671という数字が出力されるので、「およそ2/3なんだろうな」と予想がつく。その予想がついてからコードを見ると、「ハズレなら、残りは正解」という記述に気づく。チェンジ派は、最初の選択でハズレの箱を選んだ場合、残りの二つの箱のどちらかにアタリがある。司会者は残りの二つの箱のうちハズレの箱を開けるのだから、開いていない方の箱は必ずアタリである。したがってチェンジ派は「最初に選んだ箱がハズレなら景品がもらえる」ことになる。最初に選んだ箱がハズレである確率は1/3であるから、二番目に選んだ箱がアタリである確率は残りの2/3である。

また、キープ派が当たる確率が1/3、チェンジ派が2/3であることから、「キープ派とチェンジ派の当たる確率を足すと1になるだろう」と気づく。同じ条件において、キープ派とチェンジ派のどちらかは必ず当たるのだから、キープ派が当たる確率が1/3なら、チェンジ派が当たる確率は残りの2/3になるはずだ。

このように、シミュレーションの結果を見ると「後出し」で理屈が見えてくる。このあたりはリボ払いと同じである。
